# Module 1 - Login

## Prompt Version 1

Generate functional test cases in Gherkin format for the Login module of a web-based test management application.

Cover:
- Valid login
- Invalid credentials
- Forgot password
- Session expiry
- Brute-force lockout

## Prompt Version 2

Act as a Senior QA Automation Engineer.

Generate comprehensive Gherkin test scenarios for the Login module of a web-based test management application.

Include:
- Valid login
- Invalid username
- Invalid password
- Empty username
- Empty password
- Forgot password
- Remember me
- Logout
- Session timeout
- Brute-force lockout
- SQL injection
- XSS input validation
- Field validation
- Error message validation
- Password masking

Requirements:
- Use standard Gherkin syntax.
- Include Feature, Scenario, Given, When, Then, and And where appropriate.
- Cover both positive and negative scenarios.
- Include realistic edge cases.
- Avoid duplicate scenarios.
